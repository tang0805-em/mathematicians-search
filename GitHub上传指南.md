# 📤 GitHub上传指南 - 修复师生关系和图片显示

## 问题分析

你的网站上没有显示新增的师生关系，原因有两个：

### **问题1：数据库初始化只在本地运行** ❌
- 当前app.py在`if __name__ == '__main__':`块中初始化数据库
- 这个代码只在直接运行`python app.py`时执行
- 但Render使用gunicorn部署，不会执行这个块
- **结果**：Render上的数据库从不初始化，仍使用旧数据

### **问题2：维基百科图片URL被阻止** ❌
- 直链访问维基百科图片返回403 Forbidden或404错误
- 前端下载图片失败，所以显示不了
- **解决方案**：删除所有外部图片URL，让前端使用自动生成的头像

## ⚡ 常见问题

**Q: 上传后网站还是没有更新？**
A: 检查Render的部署日志：
1. 登录Render Dashboard
2. 点击你的应用
3. 进入 "Events" → 查看最新部署是否成功
4. 如果失败，查看 "Logs" 了解错误

**Q: 图片为什么还是看不到？**
A: 这是正常的，因为我们删除了不工作的图片URL。前端现在使用自动生成的头像（首字母圆形背景）。

**Q: 需要删除data文件夹中的旧数据库吗？**
A: 是的！Render上的旧数据库会在：
   - 应用重启时
   - 或第一次调用 `initialize_database()` 时
   - 自动重新创建
